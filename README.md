# 모바일(App) 피그마 레이아웃 변환 서비스

## 🔎 프로젝트 소개

• 이 프로젝트는 모바일 앱의 런타임 렌더 트리를 분석해, 개발 이후의 화면을 편집 가능한 Figma 디자인으로 자동 변환함으로써 초기 스타트업의 개발–디자인 역방향 워크플로우를 자동화하는 시스템을 구현하였습니다.

----

## 🔖 프로젝트 배경

• 최근 노코드(No-Code) 도구와 생성형 AI의 발전으로 1인 개발자가 빠르게 모바일 앱을 구현하는 흐름이 확산되고 있습니다.<br>
• 하지만 MVP 검증 이후 디자이너가 합류할 경우, 이미 구현된 화면을 Figma로 다시 제작하는 반복 작업이 필연적으로 발생하며 이는 디자인 부채와 협업 비효율을 초래합니다.<br>

----

## 🚀 유저 시나리오
### 1️⃣ 디자인 data JSON 추출<br>
• 사용자는 VSCode에서 Flutter 애플리케이션을 디버그 모드로 실행 중인 상태에서 원하는 화면으로 이동한다.<br>
  1. 익스텐션 실행: VSCode 명령 팔레트를 열고 Flutter: Extract UI to Figma 명령어를 입력
  2. 디자인 데이터 추출: 명령어가 실행되면, 내부적으로 런타임 크롤러가 현재 화면의 RenderObject 트리를 순회하여 JSON 데이터를 생성
  3. 시스템 알림: 생성된 JSON 데이터는 별도의 파일 저장 과정 없이 사용자의 클립보드에 자동으로 복사되며, "UI Data Copied to Clipboard"라는 알림 메시지를 통해 완료 사실을 사용자에게 알림

### 2️⃣ 디자인 시각화 (Design Visualization)<br>
• 사용자는 Figma로 창을 전환하여 작업을 원하는 페이지를 연다.<br>
  1. 플러그인 실행: Figma의 리소스 탭에서 본 프로젝트의 플러그인을 실행
  2. 디자인 데이터 입력: 플러그인 UI 내의 [Import from Clipboard] 버튼을 클릭하거나, 지정된 단축키(예: Ctrl+V)를 입력
  3. 렌더링 완료: 플러그인은 클립보드에 저장된 JSON 데이터를 즉시 파싱하고, 3.3절의 매핑 로직을 수행하여 Figma 캔버스 위에 실제 디자인 레이어(Frame, Text, Image 등)를 생성

## ✔️ 결론
### 1️⃣ 기대효과<br>
• 본 시스템은 개발 이후 디자인 자산을 수작업으로 재구성해야 했던 기존 워크플로우를 자동화하여,MVP 단계에서 구현된 다수의 화면을 저비용·고효율로 Figma 디자인 자산으로 전환할 수 있게 합니다.<br>
• Code-to-Design과 Design-to-Code를 연결함으로써, 개발–디자인 간 양방향 협업 생태계를 구축할 수 있습니다.<br>

### 2️⃣ 향후 연구 방향
• 향후에는 세션 기반 아키텍처를 도입하여, 코드 재주입 없이 연속적인 화면 추출이 가능하도록 시스템을 개선할 예정입니다.<br>
• 이를 통해 사용자는 화면 녹화와 유사한 방식으로 다수의 UI를 추출할 수 있으며, 시스템은 불필요한 반복 연산을 제거해 자원 효율성과 안정성을 동시에 확보할 수 있을 것입니다.<br>
